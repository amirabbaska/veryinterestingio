  • Mathematics Is there a formal way of deciding what fraction of a game is chance?

    NEW
    Q:

    The card game of "war", in which players flip cards and compare their values without ever making any decisions is clearly entirely probabilistic, (each player will win half the time, as long as the deck is perfectly shuffled before each game.) The game of chess has no probability involved because the game state is entirely dependent on input from the players. These are examples of games at the extreme ends of the scale, but is there a way of assigning a specific value of how much probability plays a part in a game for those that are in between the extremes?

    HarlequinNight A:

    This is just an aside I thought you might enjoy. Richard Garfield, the creator of Magic the Gathering, often talks about the importance of having some random elements because it allows losers to blame it on bad luck, and winners to claim it was all skill. This makes it easier to keep playing and not feel bad for losing too much.

    He goes on to describe two hypothetical games to show extreme version of combining skill and luck. The first is Rando-Chess. In Rando-Chess both players play an entire game of Chess. At the end they roll a 6 sided dice. If it is 3 or higher, the winner of the Chess game wins, otherwise if it is a 1 or 2, the loser of the Chess game wins. Basically the entire skilled chess game is a preface to having slightly better dice odds.

    The second example is "guess the nth digit of Pi". It is a totally probabalistic outcome to ask what the 11 millionth decimal of Pi is, but most people just guess a random number.

    bluesam3 A:

    Some random thoughts (initially restricted to 2-player games, but easy enough to generalise, I think):

    Consider a game G between two players. Player A takes a random decision, distributed evenly from all possible available moves. Player B always takes the move that gives them the greatest probability of winning the game in every situation. Define the chance fraction C(G) of the game to be 2P, where P is the probability that player A wins. In a perfectly random game, Player A will win exactly 50% of the moves (since there's no better decision than random chance available), so C(G) would be 1. In a game with zero chance, player A will win something extremely close to 0% of the games (that is: exactly 0 unless there exists a winning strategy for A from the start position, in which case it's the chance that A accidentally plays that winning strategy).

    Advantages: Works, pretty unambiguously determines the level of chance in the game. Can be approximated by having the best players available play against random steppers. Matches intuition of what it should be.

    Disadvantages: Ludicrously difficult to calculate precisely for games that aren't extremely simple.

    sirgog A:

    Never seen it formalized, but the best measure is the standard deviation in tournament equity given a set prize structure and a mixed skill playerbase.

    Example - 64 players bet$1 each, winner takes all, single elimination.

    If the game is pure chance, each player's tournament equity is $1. 1 in 64 chance to win.

    If the game is pure skill, for example an arm wrestle best of 17 where players are fully rested each round, the strongest person has equity of $64 and everyone else none.

    etotheipi1 A:

    One way to go about it would be to set up a round-robin tournament and then measure its non-transitivity: number of 3-cycles (i.e. A beats B, B beats C, C beats A) in the tournament.

    If we have a height-comparing competition, then the taller person would always beat the shorter person, so the transitivity of the tournament would be 0. However, if we have a rock-paper-scissors tournament with N people, you would expect to see roughly N^(3)/24 3-cycles. Games with some luck and some skill would fall somewhere in the middle.

    I once gave a talk on this graph theory topic and got this question: "couldn't we use this to measure how certain sports are more luck based than others?"
