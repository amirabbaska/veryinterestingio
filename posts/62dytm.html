    Q:

    According to logic, if division is splicing a number into equal parts, dividing by zero would be 0 parts, and the result would be 0, right?

    934has0a0fsj9s A:

    Instead of thinking of the act of dividing, let's think about what things are like after you divide. So if you divide 15 by 3 you end up with 5. Why did you end up with 5? Because 5 groups of 3 makes 15. If you divide 8 by 4 you get 2 because 2 groups of 4 makes 8.

    So what if we divide 10 by 0? You're thinking we should get 0. Do 0 groups of 0 make 10? Hmm...Weird... Is 8 divided by 0 also 0? So 0 groups of 0 also make 8? I thought 0 groups of 0 was 10?

    Essentially, division is the opposite of multiply. So think of division as asking about multiplication:

    8/4=2 because 2*4=8

    15/3=5 because 5*3=15

    10/0=0 because 0*0=10? No that's not right...

    10/0 asks the question "what times zero equals ten?" Which is just a question that has no answer.

    But here's where things get interesting, what is 0 divided by 0? You might think it's 0... Let's check:

    If 0/0=0 then 0*0=0? Yes! That works!

    But I might say that if you divide something by itself you always get the number 1. Let's check that:

    If 0/0=1 then 1*0=0? Yes! That's also right...

    If 0/0=7 then 7*0=0? Yes

    If 0/0=54 then 54*0=0? Yes

    It certainly seems like every number is going to work...

    So when you divide 0/0 you are asking a question where the answer is "all numbers". This causes problems when we work with calculations so we prefer to say that there's no answer to that question.

    Other times, when you divide 7/0 you're asking a question that simply has no answer.

    vvsj A:

    The real numbers with + and form what is called a field. A field is basically just a set of stuff which obeys the same properties ordinary real numbers do (like distribution of over + for example like FOIL). It is the generalization of the reals.

    A field must have at least two distinct objects: an additive identity (so an element, let's call it 0, such that a + 0 = a for all elements a in the field) and a multiplicative identity (a number, let's call it 1, such that a*1 = a for all elements a in the field).

    The inverse of a number x is a number y such that x*y = y*x = 1.

    To divide by something means you are multiplying by its inverse. So for example, we can divide by 3 because there is nothing wrong with multiplying by 1/3. And, of course, 3*(1/3) = 1.

    What happens if we allow 0 to have an inverse? Call its inverse b. So, if b is the inverse of 0, then this means that 0*b = 1.

    Why is this an issue? Well, because 0*anything = 0. Why is this true?

    Since 0 + 0 = 0, it is true that r*(0+0) = r*0 for any number r.

    Distribute r over the bracket like we're allowed to and you get r*0 + r*0 = r*0.

    Now, just subtract r*0 from both sides and you see that r*0 = 0 for any number r.

    So if you allow 0 to have an inverse called b, then you have two conflicting statements: 0*b = 1 but also 0*b = 0. So we can't do that. This contradicts both of the rules we laid out, and it also "proves" that 0 = 1, which is contrary to our assumption that a field must contain at least two distinct elements 1 and 0. We would have a bunch of nonsense statements in math, like 0 = 3*0 = 2*0 = 0, dividing by 0, we'd have 3 = 2.

    So we don't allow division by 0 in a field like the real numbers.

    SilentHorizon A:

    In your example, the answer is not actually 0. This is most easily shown through a quick exercise.

    Let's start with the number 1 and we'll divide it by smaller and smaller numbers to see what happens as we get closer to 0.

    1 / 1 = 1
1 / 0.5 = 2
1 / 0.25 = 4
1 / 0.1 = 10
1 / 0.0001 = 10,000

    Clearly, we can see that as we divide by smaller and smaller numbers, the result gets bigger and bigger. By this pattern, we can presume that dividing by 0 will give us a result of infinity (∞).

    However, we can approach zero from the other side as well: from the negative side. Let's start at -1 and divide by smaller and smaller numbers and see what happens.

    -1 / 1 = -1
-1 / 0.5 = -2
-1 / 0.25 = -4
-1 / 0.1 = -10
-1 / 0.0001 = -10,000

    Uh oh. This pattern makes it look as though dividing a number by 0 will give a result of negative infinity (-∞). How can something equal both infinity and negative infinity? Easy: It can't. This is why we call anything divided by zero as being "undefined." It has no meaning.
