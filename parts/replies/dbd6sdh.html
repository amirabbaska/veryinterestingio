      I'm not in the field anymore (and was just barely back "in the day") but I'll try to respond to you.

      If we define "hacking" simply as "exploiting a system to perform in a new desired way" we can include Trojan attacks (malware and the like), but there are also the classic DDOS (Denial of Service) where you bombard a system so much it basically locks up, as well as redirects (looking in the wrong place), you can include any form of phishing as hacking, and you can also include hardware hacks as well as software, a classic being the pay phone hack, or a delay/repeater device which can interrupt an on/off signal to allow undesired actions to happen in the interim.

      When it comes to security (and my experience) the weakest point is usually the user (phishing/trojan), followed up by the network (ddos, wifi interceptor, RF reader, thumbnail/CD auto exe). Guessing a password is usually hard (or should be), same with decrypting access. There are always "backdoors" and "shortcuts" in the code from the development stage. I'm sure it is possible to eventually remove all of those but a lot of them are access points for other software to engage to.

      There is a lot to it and it is outside my bailiwick now mostly (thank god) and I'm not the most current resource, but I still try to pay somewhat attention to it.
