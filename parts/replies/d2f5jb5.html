      Idk about the binning process for memory, but I doubt anyone intends faulty memory to hit the market. I suspect there is a yield test threshold for each individual chip ie "how fast can this chip go before errors occur? OK, build drives from similarly rated modules."

      It's also important to note that memory is organized in a way that lets it communicate almost directly with other memory and microprocessors. The technology makes use of (and requires) this uniform structure. One bad set of addresses would require an overhaul of how the data is handled to avoid losing it, and/or a lobotomy of large parts of the drive, neither of which is easy or worthwhile.

      There are many reasons for over provisioning, but mainly it's to give your drive the room it needs to organize your files efficiently. One example is for large file operations: they will obviously go faster if spread evenly across every lane that can deliver data. During a write, if data is "in the way," your drive should probably move it. Limited space might require a large number of poorly distributed reads and writes to "shuffle" this data out of the way. Both doing and not doing this result in decreased performance and drive life, all for want of space.
