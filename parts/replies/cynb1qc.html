    A webpage is actually hundreds of websites that needs to be accessed

    Well, not really hundreds of websites, but hundreds of different resources. I don't think the usual website uses anywhere near 100 other websites. Most likely it'll use some librairies of code from Google and stuff like that, but most of the resources are from the same 'website'.

    It's more like:

    Your browser: "Hey web server, give me the home page"

    The server: "Here's the HTML of the home page"

    Your browser, while reading the HTML: "Hey website, I need this CSS file", "Hey website, I need this picture", "Hey website, I need this JavaScript file", etc etc... Each of these is a different request / connection to a web server.

    Then, it's possible that the browser will need other resources coming from the CSS and JavaScript execution.
