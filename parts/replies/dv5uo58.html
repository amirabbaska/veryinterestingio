      There are a lot of advantages to cout over printf:

      1. You can write anything to cout (provided you define operator<<). You can only write fundamental types using printf.

      2. Outputting to any stream (file, console, stringstream) is identical (all use operator<<). Whereas if I want to write to file I need to use fprintf, to string I need to use sprintf. Even writing to stderr needs to use fprintf which has different arguments.

      3. printf is not type safe, operator<< is.

      4. sprintf has the buffer size problem (need to guess a buffer size before using it). ostringstream doesn't have that issue.
