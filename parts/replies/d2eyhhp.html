    • Koooooj

      It's not a direct cause-effect relationship, but so far we haven't come up with memory that is both fast and non-volatile. For system memory we want the absolute fastest memory we can get, and that memory happens to be volatile.

      That said, though, there have been various attempts at using RAM for longer-term data storage. The simplest is just to make a RAM Disk, which is a software construct that mounts a file system on your computer where the contents of that file system are always stored in RAM (and perhaps stored on a hard drive for when the computer turns off). For programs that are constrained by how fast they can read data this can be a good tool, although of course you still have to get the data into RAM in the first place. This is mostly for programs that have a lot of back-and-forth to disk, where you don't want to re-write the program to work out of a larger amount of RAM than it was originally envisioned for.

      Another approach is to actually mount RAM modules on a PCI card and use that as a disk. I haven't seen one of these in a long time. They would often even come with a specific power port so that the RAM could store the data while the rest of the computer is turned off.
