    I don't understand how the assumption that no algorithm exists is made. As far as I understand it's just incalculable; in order to solve M you must solve M. I imagine it more like an infinite loop rather than proof M cannot exist. Plz help

    • thomaswdyoung

      In order to prove a negative statement (e.g. "there is no algorithm that always tells us whether any given mathematical statement is true or not") we usually start by assuming the opposite (i.e. "there is an algorithm") and show that this assumption leads to a contradiction (i.e. "the statement is both true and false").

      As an example of this reasoning, suppose we want to prove "not all birds can fly". We start by assuming that all birds can fly. An emu is a bird. Therefore, our assumption says an emu can fly. But an emu cannot fly - a contradiction. Since our assumption leads to a contradiction, we can conclude that the opposite is true: not all birds can fly.

      So in the argument that the algorithm does not exist, we do not assume that no algorithm exists. We assume that some algorithm exists (which we refer to as M) and prove a contradiction.

    • thomaswdyoung

      The halting problem is how Turing solved the problem. For that, we suppose we have an algorithm for the "simpler" problem of determining whether a Turing machine eventually halts or runs forever. From such a machine, we can construct another machine that will halt exactly if the original machine says it won't.
