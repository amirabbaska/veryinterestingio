    • Mr-Magnus

      a thing to remember is that an important consequence of storing memory as a long string of uninterrupted binary is that the computer never knows what the information represents. It is (usually) never stored where the information starts and ends and how to parse it, what it means. a string of binary is just a string of binary, that text file is just a string of binary, that .exe is just a string of binary, the computer sees no difference. it lacks context.

      It's the job of the software (or the CPU architecture) to make sense of the information it requests. Like what happens in the example OP gave: The text editor knows that it is used to parse text, it is programmed with rules to understand memory as f.i Asccii text. so, anything it receives is interpreted that way, be it right or wrong.

      The CPU assumes machine code, it understands machine code and gives that as context. But pass it a text file and it will throw an error, the information you passed it is meaningless under the context of machine code.

      It's the software that translates the binary received, not the computer, if that makes sense. A chinese book is understood by a Chinese speaker, even if the library does not understand it.
