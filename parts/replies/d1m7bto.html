  • FreeTekno

    I develop procedural 3d art for video games and we use a "controlled random seed".

    It means that the stuff we "randomly" generate is actually controlled by a number that we assign as an input to the tool so that we can always avoid having duplicate or repeating results as well as always be able to go back to a randomly generated result.

    Edit: Pretty much like this. The number of the random seed will always generate the same results, but because the random seed has no limit the results can always be unique. However with complex tools with multiple random seeds/inputs its hard and sometimes even impossible to go back to previous results, but atleast they never repeat.
