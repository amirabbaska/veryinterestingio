        Because the wireless Telecom standards that we use define symbol length in time, they will have the same throughout. However, if you don't define the symbol length as a fixed length of time and instead leverage the increased number of cycles per unit of time and instead define symbol length as the time for a set number of cycles.

        So for LTE the symbol length is 66.67us, but at 700 MHz a single cycle would only take 1.42ns, but a single cycle for 2100Mhz is 0.476ns. So if you defined the symbol length in time as a cycle count you could theoretically have 3x the tput on 2100Mhz as compared to 700Mhz.

        This comes with challenges, having a single defined symbol length makes timing and coordination easier, as well as keeps the Baseband abstracted from the transmit frequency.
