      Today's programmers completely could do that. But they have other priorities, because it really doesn't matter for the use cases that people care about.

      I can make my program smaller -- or I can implement three new features a customer wanted. Or I can make it compatible with more devices. Or I can make it look prettier.

      My favorite saying about software development is "Software is never released -- it escapes." There is no hard, functional limit to how much you can keep improving the code for any reasonably complex bit of software. But who is going to pay you to do that? At some point you have to ship it.
