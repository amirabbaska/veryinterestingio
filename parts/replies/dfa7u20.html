      Shit can slip on very easily if you don't have it

      Not if you do as I said: use a firewall, install updates, don't click on shitty links.

      If you can find an antivirus suite which does nothing other than explicit scans, which does not install any drivers or browser plugins, does not MITM your TLS traffic, and so on, then yes, running its scans every once in a while is probably more or less harmless.

      But if it does any of those things, then you are increasing your attack surface. And yes, Comodo does install a filter driver to process any and all I/O passing through. Which means that, if there is a vulnerability in that driver, simply downloading the right file, without even opening it, may compromise your computer. Exactly like happened here.

      Whether it slows down your computer is really a secondary concern. The bigger issue is that it makes you more vulnerable to attackers.

      Both Chrome and Firefox devs have gone on record to say that AV software is the single biggest impediment that stops them from shipping a secure browser.

  • Strabe

    Performance is not the primary goal of antivirus software, but one of the main competition points is performance - so they do care.

    Calling all antivirus software "crap" is a crap statement. The reality is that writing antivirus software is (much, much) harder than writing a virus.

    Minimizing "risky behaviors" such as clicking risky links helps, but there have been bugs in image libraries that have been exploited before. Keep some sort of antivirus active, and for your mother's sake, do backups.

    Most good security suites also include backup, which is an essential part of security anyway.

      but there have been bugs in image libraries that have been exploited before

      Yep. And AV software often hasn't caught it. There have also been bugs in AV software that has been exploited before. Many times.

      I already responded to another commenter with some sources to back up what I'm saying, but I'll at least drop this excellent article, which quotes some actual security experts.

      One of the key ways to achieve security is to minimize your attack surface. The less software is installed, and the less software has privileged access to your computer, the better. And AV software is horrible because it runs with full privileges, and often even installs kernel-mode components. And they tend to install browser addons and filter drivers to process any and all I/O, and often even install custom certificates to man-in-the-middle your presumed encrypted internet traffic. And vulnerabilities have been found in all of those components at one time or another, leaving the users more vulnerable than if they hadn't run AV at all.
