      The gif format was supposed to be a method to send several pictures in one package so if you wanted to send ten pictures, you'd send just one file instead of ten.

      The thing was that, if you plaid them sequentially, you could have animation, so in the early web days they started using them for that and they stuck.

      There are some more technical issues at play, like for instance, is technically more complicated to insert a video than a gif. Also gifs are always gifs, videos can be mp4, webm, ogv, etc. Mobile devices usually don't autoplay videos but do gifs. Having multiple videos reproducing at the same time in the same webpage can give problems too. These all are things that are being ironed right now in current generation browsers and devices, but it's taking a while.

      But deffinately yeah, gifs are not only behind the curve, they never were intended for this purpose on the first place.
