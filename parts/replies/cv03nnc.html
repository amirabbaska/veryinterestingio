    Programmer here as well. What I started doing a while back, is using an adaptive loading bar in my splash screens. The way it works, is the first time the program is run, I use the usual method of computing remaining loading time. At the same time, the program measures how long the loading actually took, and saves that information.

    The next time the user starts the program, I average calculated time with actual time. I also record the time it took to start the program yet again. That way, I end up using a running average, of last N load durations (usually 10), which is pretty damn accurate at that point, and allows me to present the user with a smooth loading bar.

    If I'm in a rush, I even skip the loading calculation time in step one, and just use a hardcoded value we get from testing in-house. After that averaging is done as described above, with actual loading times.
