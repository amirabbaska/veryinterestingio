• Mathematics What is the mistake in this mathematical proof?

    NEW
    Q:

    From Hexaflexagons and Other Mathematical Diversions by Martin Gardner

    a=b+c

    multiply both sides by a-b

    a^2 -ab=ab+ac-b^2 -bc

    move ac to the left

    a^2 -ab-ac=ab-b^2 -bc

    factor:

    a(a-b-c)=b(a-b-c)

    divide both sides by a-b-c

    a=b

    He says that this is a fallacy, but can't you just say that c=0?

    Rannasha A:

    In the last step, you divide both sides by a - b - c, but this operation is only valid when a - b - c is not equal to zero. But from the very first assumption, we have that a = b + c, so a - b - c = b + c - b - c = 0.

    The derivation is correct up to this point, but in the second-to-last step, it actually says: a 0 = b 0, which is obviously true, but can't be used to infer that a = b.

    You start with

    A = B + C

    Subtract both sides by (B + C)

    A - B - C = 0

    So later when you divide both sides by (A - B - C) you're actually dividing by zero, which you can't do, so everything gets kinda screwed up.

    TheSaddestGrape A:

    Division by zero. It's nearly always division by zero. Go over every step where division occurs and check if the denominator is zero (you've got a pretty good cue right there: "divide both sides by...")

    Paradox-producing errors might also occur if you're not careful with square roots or logarithms. Checking your domains and ranges will usually root out those errors.

    Zarigis A:

    As others have said, the proof error stems from the final division step, which is undefined if a - b - c = 0. Your specific question indicates a deeper misunderstanding though. It is a fallacy because of the implicit quantifier: that "a=b" is apparently true for all a,b, and c such that a=b+c, not some particular values of them (obviously true if c=0, as you say). The reason it is a fallacy is because this is clearly absurd for many values (although one counter-example is sufficient).

    Drachefly A:

    IN GENERAL, when you encounter a totally false proof like this, just put in arbitrary non-special numbers and see where you reach the first false statement.

    Here, it could be 3 = 2 + 1 (both sides equal 3)
    3*3 - 3*2 = 3*2 + 3*1 - 2*2 - 2*1 (both sides equal 3)
    3*3 - 3*2 - 3*1 = 3*2 - 2*2 - 2*1 (both sides equal 0)
    3*(3 - 2 - 1) = 2*(3 - 2 - 1) (both sides still equal 0)
    3 = 2 (NOPE)

    For false proofs that have only special cases wrong, it would take a special case to discover it.
