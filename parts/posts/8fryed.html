    CcntMnky A:

    Both. Within one technology generation, there will be multiple capacities by stacking (inside the package) or grouping (packages on a board). The technology investment is to achieve a next technology that does the same capacity in half the space, allowing to double. This is usually by fabricating the same transistor in less area.

    garrett_k A:

    I've worked in the storage industry, though not specifically on this type of device, so I might be missing a few minor details. Hopefully this will help, though.

    A quick search found this article which is focused on high-performance programming. SSDs are very similar in relevant ways to USB sticks.

    In general, there are 3 major components to a flash drive (or even SSD). There's the raw flash chip(s) themselves. There's the flash translation layer which is what makes the flash chips look and act like a drive. And then there's the interface controller which provides USB connectivity (or SATA connectivity, or whatever).

    When it comes to the engineering of these devices, there are lots of things to be considered including assembling costs, defect rates, market desire and willingness to pay.

    Using more than 1 flash chip means you have to pay more for additional assembly costs, if only the extra machine time to install it.

    The flash translation layer is also designed to handle a certain number of flash chips. (I'm assuming it's a separate IC, but SoC design keeps changing things). In general, supporting more chips means more connections which means more silicon and a larger packaging. If you can support a lot of flash chips you are likely able to support higher throughput (which is why SSDs are usually much faster than flash drives even for similar capacities). But if you're trying to go for cost minimization, you might use a shared bus if the flash chips support it and you're willing to take on additional debugging headaches.

    The interface layer probably isn't that interesting.

    If I was tasked with designing a USB memory stick of a particular capacity, I'd probably check to see if the cost of two smaller flash chips plus the cost of a translation-layer chip which supported two flash chips, plus costs of manufacturing, etc., were more or less than the cost of a single larger flash chip with the simpler translation layer, etc. And then do that. Because in this space customers seem to care almost exclusively about price and capacity to the exclusion of all else.

    jnecr A:

    There is a limit to the largest module of RAM that can be produced. And that is limited by the largest individual DRAM chip and the largest "rank" that a system can handle. The largest DDR modules are simply two smaller modules packed together onto the same PCB.

    The only 128GB DDR4 modules I could find are "octal-ranked." I.e. they are 8 individual modules placed on the same PCB. 64GB modules are commonly "quad-ranked," however, older ones could also be octal-ranked.

    So, why don't we have 16-ranked modules? Because the systems that this RAM runs on is generally limited to 8 ranks per memory channel. So, if you use a 128GB module, you cannot put any other modules in that memory channel. Server systems frequently have 3-4 memory slots per channel. So, you could either configure your system with a single 128GB module or with two 64GB modules, or with four 32GB modules. The 32GB modules will be the cheaper option and therefore in servers where space is not limited the multiple module approach would be preferred.

    But, not all systems are large enough to allow for 4 modules/channel. Which is why we have an octal-ranked 128GB DDR4 RAM module.

    thephantom1492 A:

    A memory stick can be made in several ways.

    1- It can be a single or multiple chips. If you open some usb keys you can see that only one side have a chip, the other may only have solder pads. For example, a 32GB could have one side used while the 64GB will have both.

    2- Some other will stack vertically some chips.

    3- You can take two or more dies (that's the internal of the chip) and put it in the same package.

    4- you can take the circuitry and double it then produce the die.

    5- you can stack the circuitry in the die (which is basically what 'vertical NAND' is).

    6- you can use a smaller process and shrink each cells, and fit more in the same space.

    And you can also use more than one of those. Each have their advantages and disadvantages. For example, adding two chip allow for a single design, single bill of material, but two sizes. You order in bigger quantity and get a better price on some parts, get a single setup for everything. All you have to do is tell the machine to not install the second chip. The overall price however is slightly higher as you now have 2 chips, and a single smaller one may be cheaper. A two chip solution can be faster (you can spread the read/write across both, and adding the speed), which also cause the smaller capacity to be slower.

    A smaller process is initially more costly, and there is more reject initially. Eventually it become cheaper as they fine tune as they use less materials.

    Jigabit A:

    Answering your question about the stick being so much bigger than the actual chip inside. Many things to consider there. 1) that's not always true. There are some wickedly small and still high capacity memory sticks. 2) you have to consider that it's not just the memory chip inside. There is some power cirquitry and a controller chip also. The more actual memory modules you stick in there the more complicated control gets (you may need a bigger control chip) and also power cirquitry gets more complicated. For this reason a single higher capacity memory module is better but it's more costly. 3) I think this is done for ergonomic reasons as well. You could make the physical stick smaller but should you? It becomes easier to lose and harder to unplug/plug. So it will cost more and ultimately not actually be better, potentially worse actually, from a usability point of view

    Edit: to answer the original question about gluing modules together. Yes this happens all the time. Look at a ram stick. It says "8 gb" but has like... 12? Ish chips on it that make up 8gb total. Look at a video card. It says it has 4gb of memory but that too is spread across multiple memory chips. Ssds do the same. So yes you can make "the biggest memory stick ever" exactly how you Invision it. Just unlikely that it will be that small. Also memory sticks (unlike other things I've mentioned here) draw their power form the USB port which will very much limit how much "things" you can have on the stick.
