If we have a^(b)=b^(a), then taking logs and rearranging gives log(a)/a=log(b)/b. So if we look at the function f(x)=log(x)/x, then we'll want to find two different values x=a and x=b so that f(a)=f(b). If f(a)=f(b), then we can just work backwards to a^(b)=b^(a).

So what does f(x) look like? Well, it's negative for 0<x<1 and positive for all x>1. Furthermore, looking at it's derivative, you can find that it is increasing when 0<xe. And, finally, the limit of f(x) at x=infinity is zero. So, f(x) is increasing whenever it is negative and this means that it never repeats a negative value, since it is only going up when it is negative, and once it's positive it's never negative again. So we can't have f(a)=f(b) when either a<1 or b<1. But, since it is increasing from x=1 to x=e, and f(1)=0, and it is decreasing from x=e onward, and f(x)->0 as x->infinity, it follows that for any 1<a<e, there is exactly one b>e so that f(a)=f(b). So there are infinitely many solutions to this.

Here is a plot of how this works

Note that 2 is the only integer between 1 and e, so a=2 and b=4 is the only nontrivial integer solution.

We can actually parameterize all of our solutions. In particular, if r is any real number, then maybe we can find an 1<a<e so that b=a^(r) gives us a solution? Plugging b=a^(r) into log(a)/a=log(b)/b and rearranging things, gives a=r^(1/[r-1]) so we'll have b=r^(r/[r-1]). Since any 1<a<e can be written as a=r^(1/[r-1]) for some real number r>1, this means that every solution looks like this. So, pick any r>1 and

  • a=r^(1/[r-1])

  • b=r^(r/[r-1])

will be a solution to the equation a^(b)=b^(a).

But what about rational solutions? If there's no guarantee that when x=a is rational that the corresponding x=b will also be rational. When does this happen? It turns out that it will happen infinitely often. In particular, if n is any positive integer, then

  • a = (1+1/n)^(n)

  • b = (1+1/n)^(n+1)

will work. This works by making r an rational number so that 1/(1-r) and r/(1-r) are integers, and plugging it into our formula. In particular, picking r=1+1/n works. The first few rational solutions are

  • a=2, b=4,

  • a=9/4, b=27/8

  • a=64/27, b=256/81

To see why this form provides all the rational solutions requires some work, but it isn't too difficult, see this article for details.

existentialpenguin A:

If we plot x^y = y^(x), we see that the graph has two pieces: a straight piece and a curved piece that intersect at the point (e,e).

The straight piece is just y=x; this corresponds to the trivial solutions.

The curved piece is given by y = -W(-log(x)/x) x / log(x), where W is the principal branch of the Lambert W function. W(x) is a transcendental number whenever x is algebraic and nonzero, but as /u/functor7 explains, this doesn't prevent -W(-log(x)/x) x / log(x) from being algebraic or even rational for rational values of x.
